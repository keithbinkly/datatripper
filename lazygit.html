What is it?

lazygit is a terminal UI for Git that makes staging, committing, branching, and rebasing fast and visual. Instead of typing git commands, you navigate panels and press single keys.

It's especially powerful for interactive rebasing, cherry-picking, and managing complex git histories.

Installation

brew install lazygit

# Launch (from any git repo)
cd your-repo
lazygit

Interface Panels

The UI has 5 main panels. Switch with 1-5 or Tab/Shift+Tab:

1 - StatusRepo status, upstream info
2 - FilesUnstaged/staged changes (most used)
3 - BranchesLocal branches, remotes, tags
4 - CommitsCommit history (rebasing happens here)
5 - StashStashed changes

Basic Workflow

# In Files panel (2):
j/k           # navigate files
space         # stage/unstage file
a             # stage all
c             # commit (opens editor for message)
shift+C       # commit with editor for message

# Quick commit flow:
space space space   # stage files
c                   # commit
type message
enter               # done

Staging Hunks

You can stage individual chunks of a file, not just the whole file:

# On a file with changes:
enter         # expand file to see hunks
space         # stage/unstage current hunk
a             # stage all hunks in file

# Line-by-line staging:
enter         # on a hunk, expand to lines
space         # stage individual line

Branch Operations

# In Branches panel (3):
space         # checkout branch
n             # new branch
d             # delete branch
M             # merge into current
r             # rebase current onto selected
f             # fetch
shift+P       # push
p             # pull

Commit Operations

# In Commits panel (4):
enter         # view commit details/diff
r             # reword commit message
d             # drop commit (in rebase)
e             # edit commit (in rebase)
s             # squash into previous
f             # fixup (squash, discard message)
shift+C       # cherry-pick commit
ctrl+j/k      # move commit up/down (reorder)
g             # reset to commit (soft/mixed/hard)

Interactive Rebase

This is where lazygit really shines. Much easier than command-line rebase:

# To rebase last N commits:
# 1. Go to Commits panel (4)
# 2. Navigate to the commit BEFORE where you want to start
# 3. Press 'e' to start interactive rebase

# During rebase:
s             # squash commit
f             # fixup 
d             # drop
r             # reword
ctrl+j/k      # reorder
m             # mark as edit point
c             # continue rebase

# If conflicts:
# Files panel shows conflicts
# Edit file, then press space to mark resolved
# Press 'c' to continue

Stash

# From any panel:
shift+S       # stash all changes (with message prompt)
s             # stash (quick, no message)

# In Stash panel (5):
space         # apply stash
g             # pop stash (apply + delete)
d             # drop stash

Other Useful Keys

?Help (shows all keybindings)
qQuit
/Search in current panel
@Open command log (see git commands run)
+Expand/collapse panels
xOpen menu for current context
zUndo (via git reflog)
ctrl+rSwitch repos

Configuration

Config file: ~/.config/lazygit/config.yml

# Example config.yml
gui:
  theme:
    lightTheme: false
  showFileTree: true
  showRandomTip: false

git:
  paging:
    colorArg: always
    pager: delta --dark --paging=never
  
keybinding:
  universal:
    quit: 'q'

Integration with nvim

Many nvim users open lazygit in a floating terminal. Popular plugins:

  • kdheepak/lazygit.nvim - Dedicated lazygit plugin
  • Or just use :terminal lazygit
  • LazyVim includes lazygit by default (<leader>gg)
