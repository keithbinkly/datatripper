Neovim
Text Editor • The Big Rabbit Hole

What is it?

Neovim is a modern fork of Vim, the legendary text editor. It's keyboard-driven, meaning you navigate and edit without touching a mouse. The learning curve is steep, but once mastered, it's significantly faster than GUI editors.

Neovim improves on Vim with Lua-based configuration, better plugin architecture, and built-in LSP (Language Server Protocol) support for IDE-like features.

Installation

brew install neovim

# Verify
nvim --version

Understanding Modes

Vim/Neovim has multiple modes. This is the key concept that confuses beginners:

Normal modeDefault. Navigate and run commands. Press Esc to return here.
Insert modeType text. Enter with i, a, o, etc.
Visual modeSelect text. Enter with v (char), V (line), Ctrl+v (block).
Command modeRun Ex commands. Enter with :

Essential Commands

# Opening files
nvim file.txt          # open single file
nvim .                 # open file browser (netrw)
nvim file1 file2       # open multiple files

# Saving and quitting (command mode)
:w                     # save
:q                     # quit
:wq                    # save and quit
:q!                    # quit without saving
:wa                    # save all buffers
:qa                    # quit all

Navigation (Normal Mode)

h j k lLeft, down, up, right (basic movement)
w / WNext word / WORD (includes punctuation)
b / BBack word / WORD
e / EEnd of word / WORD
0 / $Start / end of line
^ First non-whitespace character
gg / GTop / bottom of file
{ / }Previous / next paragraph
Ctrl+d / Ctrl+uHalf-page down / up
Ctrl+f / Ctrl+bFull page down / up
%Jump to matching bracket

Editing (Normal Mode)

i / aInsert before / after cursor
I / AInsert at start / end of line
o / ONew line below / above
xDelete character
ddDelete line
yyYank (copy) line
p / PPaste after / before
uUndo
Ctrl+rRedo
.Repeat last command

The Power: Motions + Operators

Vim's killer feature is combining operators with motions. The pattern is: operator + motion

# Operators
d = delete    y = yank (copy)    c = change (delete + insert)

# Examples
dw      # delete word
d$      # delete to end of line
d3w     # delete 3 words
yy      # yank line
y3j     # yank 3 lines down
ciw     # change inner word (delete word, enter insert)
ci"     # change inside quotes
di(     # delete inside parentheses
da{     # delete around braces (including braces)
ct,     # change to comma

This composability is what makes Vim fast—you learn a few operators and motions, then combine them infinitely.

Search and Replace

# Search
/pattern           # search forward
?pattern           # search backward
n / N              # next / previous match
*                  # search word under cursor

# Replace
:s/old/new/        # replace first on line
:s/old/new/g       # replace all on line
:%s/old/new/g      # replace all in file
:%s/old/new/gc     # replace all, confirm each

Configuration

Config lives at ~/.config/nvim/init.lua (Lua) or ~/.config/nvim/init.vim (Vimscript).

Recommendation: Don't configure from scratch. Use a starter config:

LazyVimMost popular. Sensible defaults, easy customization.
NvChadBeautiful UI, fast. More opinionated.
AstroNvimFeature-rich, IDE-like out of the box.
kickstart.nvimMinimal starting point to learn from.

git clone https://github.com/LazyVim/starter ~/.config/nvim

Learning Path

  1. Run vimtutor in terminal (built-in tutorial, ~30 min)
  2. Use nvim for small edits, keep VS Code as fallback
  3. Install LazyVim for a usable setup immediately
  4. Learn motions gradually—add one new motion per day
  5. Don't customize until you understand what you're changing
